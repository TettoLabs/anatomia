## General Coding Patterns

**Follow detected patterns:** Reference scan.json and context files for detected error handling, validation, database, auth, testing patterns specific to your project.

**Use project conventions:** Follow naming, imports, and code style from conventions.md exactly.

**Study existing code:** When unsure about a pattern, find similar feature in codebase and follow the same approach. Consistency matters more than theoretical best practices.

**Ask when uncertain:** If pattern is unclear or multiple approaches seem reasonable, ask user for guidance on which pattern to follow.

**Framework documentation:** Reference official documentation for {{framework || 'your framework'}} when framework-specific questions arise.

**Avoid introducing new patterns:** Match existing code patterns. If truly need a new pattern, propose in architect mode first.
