## Detailed Findings

### Finding 1: run-check.sh hardcodes monorepo fallback path

- **File:** `packages/cli/templates/.ana/hooks/run-check.sh`
- **Line:** 36
- **Code:**
  ```bash
  if command -v ana &>/dev/null; then
    ana setup check "$@"
  else
    node packages/cli/dist/index.js setup check "$@"
  fi
  ```
- **Assumption:** When `ana` is not in PATH, it assumes CWD contains `packages/cli/dist/index.js` (the Anatomia monorepo layout).
- **Failure mode:** On external project like `/tmp/getpaid/`, the fallback fails with:
  ```
  Error: Cannot find module '/tmp/getpaid/packages/cli/dist/index.js'
  ```
  The script exits 1 and outputs an error, but the error message is confusing to users who don't know what `packages/cli/dist/index.js` means.
- **Fix suggestion:** Use `npx anatomia setup check` as fallback, or locate the package via npm/pnpm resolution.

### Finding 8: resolveWasmPath assumes monorepo layout

- **File:** `packages/analyzer/src/parsers/treeSitter.ts`
- **Lines:** 56-73
- **Code:**
  ```typescript
  function resolveWasmPath(packageName: string, wasmFileName: string): string {
    const candidates = [
      // packages/analyzer/node_modules/<pkg>/<file>.wasm
      join(__dirname, '..', '..', 'node_modules', packageName, wasmFileName),
      // monorepo root node_modules/<pkg>/<file>.wasm (pnpm hoists)
      join(__dirname, '..', '..', '..', '..', 'node_modules', packageName, wasmFileName),
    ];
  ```
- **Assumption:** The module is running from within the Anatomia monorepo structure (`packages/analyzer/src/...` or `packages/analyzer/dist/...`).
- **Failure mode:** For npm-published package installed in external project:
  - Installed at `node_modules/anatomia-analyzer/dist/parsers/treeSitter.js`
  - Path `../../node_modules` resolves to `node_modules/anatomia-analyzer/node_modules` (doesn't exist with pnpm)
  - Path `../../../../node_modules` goes outside the project entirely

  Error: `WASM file not found: tree-sitter-python/tree-sitter-python.wasm`
- **Fix suggestion:** Use proper module resolution (e.g., `require.resolve()` on the WASM packages themselves, or import.meta.resolve if supported).

### Finding 10: ASTCache writes to CWD-relative path

- **File:** `packages/analyzer/src/cache/astCache.ts`
- **Line:** 88
- **Code:**
  ```typescript

    this.cacheDir = join(projectRoot, '.ana/.state/cache');
  }
  ```
- **Assumption:** `projectRoot` is the directory where `.ana/` should exist or be created.
- **Failure mode:** During `ana init`, the analyzer is called with CWD as projectRoot. The ASTCache then creates `.ana/.state/cache/` in CWD *during analysis* (before the proper `.ana/` is created atomically). This is why init.ts:176 has to clean up:
  ```typescript

  await fs.rm(anaPath, { recursive: true, force: true });
  ```
  This is a known workaround, not a clean design.
- **Fix suggestion:** Pass explicit cache directory to ASTCache, or use temp directory during init.

### Finding 13: getTemplatesDir in init.ts checks `dist` in path

- **File:** `packages/cli/src/commands/init.ts`
- **Lines:** 833-843
- **Code:**
  ```typescript
  function getTemplatesDir(): string {
    const __filename = fileURLToPath(import.meta.url);
    const __dirname = path.dirname(__filename);

    // Check if running from dist/ or src/
    const isCompiled = __dirname.includes('dist');

    return isCompiled
      ? path.join(__dirname, 'templates') // dist/ → dist/templates/
      : path.join(__dirname, '..', '..', 'templates'); // src/commands/ → templates/
  }
  ```
- **Assumption:** The module path contains `dist` when built, and relative paths to templates work from there.
- **Failure mode:** For npm-published package:
  - Module at `node_modules/anatomia-cli/dist/commands/init.js`
  - `__dirname` = `.../node_modules/anatomia-cli/dist/commands`
  - `isCompiled = true`
  - `templates` path = `.../node_modules/anatomia-cli/dist/commands/templates` (WRONG!)
  - Correct would be: `.../node_modules/anatomia-cli/dist/templates`

  The path calculation is off by one level.
- **Fix suggestion:** Calculate relative path correctly: `path.join(__dirname, '..', 'templates')` for dist.

## Test Scenario: External Project Setup Trace

When user runs `ana init` in `/tmp/getpaid/` and then opens CC for setup:

### Step 1: `ana init` in terminal
- **Works?** Depends on how user installed Anatomia.
- If global install with `npm install -g anatomia-cli`: Works ✓
- If dev dependency: User runs `npx ana init` - Works ✓
- If running from cloned Anatomia repo: Works ✓

**Failure point:** Finding 13/14 - `getTemplatesDir()` may fail for npm users. Templates not found.

### Step 2: CC session starts, loads hooks from `.claude/settings.json`
- **Works?** Yes, CC loads `.claude/settings.json` from project root.

### Step 3: User tells CC to start setup, writer agent runs
- Writer writes context file
- PostToolUse hook fires: `.ana/hooks/verify-context-file.sh`
- Script runs: `bash .ana/hooks/run-check.sh ...`
- **Failure point (Finding 1):** If `ana` not in PATH, fallback tries `node packages/cli/dist/index.js` - FAILS

**Result:** Verification fails with confusing error about `packages/cli/dist/index.js`.

### Step 4: Writer tries self-check per agent instructions
- Runs: `bash .ana/hooks/run-check.sh [filename] --json`
- **Same failure point as Step 3**

### Step 5: SubagentStop hook fires when writer completes
- `.ana/hooks/subagent-verify.sh` runs
- **Same failure point as Step 3**

### Step 6: User runs `ana setup complete`
- If `ana` in PATH: Works for CLI call
- Reads `ENTRY.md` template from `getTemplatesDir()`
- **Failure point (Finding 13/14):** Template not found for npm users

## Conclusion

The codebase has **14 path assumptions** that break on external projects. The most critical are:

1. Hook script fallback using hardcoded monorepo path
2. WASM file resolution using relative path walks from `__dirname`
3. Template directory resolution using incorrect relative path calculation

These should all be fixed in a single pass before external project testing can work reliably.
