# Spec: Context Status Command

**Created by:** AnaPlan
**Date:** 2026-03-29
**Scope:** .ana/plans/active/context-status/scope.md

## Approach

New `context` parent command with `status` subcommand, following the `work.ts` command pattern. The command reads 7 setup context files from `.ana/context/`, checks existence and mtime, counts git commits since each file's last update, and outputs a health report. Human-readable by default, `--json` for machine consumption. Updates `.meta.json` lastHealth field on every run.

**Key patterns to follow:**
- Command structure mirrors `work.ts` (parent command with subcommand, `--json` option)
- Git operations use `execSync` with `{ stdio: 'pipe', encoding: 'utf-8' }` wrapped in try-catch
- Human output uses chalk (green for present, red for missing, yellow for stale indicator)
- JSON output via `JSON.stringify(output, null, 2)`

**The 7 setup files** (hardcoded, matches existing .meta.json tracking):
1. project-overview.md
2. architecture.md
3. patterns.md
4. conventions.md
5. workflow.md
6. testing.md
7. debugging.md

Note: `analysis.md` exists in `.ana/context/` but is generated by the analyzer, not a setup file.

**Git commit counting:** Use `git rev-list --count HEAD --since="<ISO-date>"` to count commits since file mtime. This counts all repo commits, not path-specific commits (scope explicitly chose simple approach for v1).

## Testing Strategy

- **Unit tests:** Test `getContextStatus()` function directly with temp directories
- **Integration tests:** Full command execution via captured console output (follow `work.test.ts` `captureOutput` pattern)
- **Edge cases:**
  - No `.ana/` directory (exit 1)
  - Not a git repo (graceful degradation, show file age only)
  - Some context files missing (show as "missing", don't fail)
  - All context files present
  - Context file exists but is empty (still counts as "exists")
  - File mtime in future (handle gracefully)

**Test isolation:** Each test uses temp directory with `fs.mkdtemp`, cleans up in `afterEach`. Tests that need git initialize a repo in the temp dir.

## Dependencies

- `.ana/` directory must exist (command checks and exits 1 if missing)
- `.ana/.meta.json` must exist for lastHealth update
- No external service dependencies

## Constraints

- No network calls (git operations are local only)
- Must work on Linux, macOS, Windows (use `node:path` for path operations)
- Exit codes: 0 for success, 1 for errors (missing .ana/)
- ESM imports with `.js` extensions

## Gotchas

1. **Git commands need `encoding: 'utf-8'`** or output is Buffer. Pattern from work.ts line 96: `execSync(..., { stdio: 'pipe', encoding: 'utf-8' })`.

2. **fs.statSync returns mtime as Date object** — convert to ISO string for git command and human display.

3. **Human-readable age calculation** — use simple logic: "X days ago", "X hours ago", "just now". Don't pull in a date library.

4. **lastHealth shape must match existing** — from `.meta.json`:
   ```json
   "lastHealth": {
     "timestamp": "2026-03-29T04:05:02.122Z",
     "totalFiles": 8,
     "setupFiles": 7,
     "setupFilesPresent": 7,
     "missingSetupFiles": 0,
     "staleFiles": 7
   }
   ```
   Update all fields. `staleFiles` = count of files with commits > 0 since update.

5. **Don't fail on git errors** — wrap all git operations in try-catch, return null/0 on failure. The command is informational.

6. **The 7 setup files are hardcoded** — they're a known list, not discovered dynamically. This matches the existing .meta.json `setupFiles: 7` tracking.

7. **process.exit() vs throwing** — follow `agents.ts` pattern: `console.error()` then `process.exit(1)` for user-facing errors. For testability, the exported function should throw so tests can catch it.
